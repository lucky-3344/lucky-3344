# Google Maps Tool - 地理特征分析功能说明

## 📌 新增功能

### 🆕 地理特征识别

除了区域属性判断外，现在支持识别以下地理特征：

| 特征 | 符号 | 说明 |
|------|------|------|
| **路边** | 🛣️ | 靠近道路、街道的位置 |
| **十字路口** | ✔️ | 多个方向都有POI的交叉口 |
| **高山** | ⛰️ | 山区、高原等高海拔地区 |
| **屋面** | 🏠 | 楼顶、天台、屋顶位置 |
| **水体** | 💧 | 河流、湖泊、海边等水面 |
| **绿地** | 🌳 | 公园、广场、绿地、森林 |

---

## 🎯 使用方式

### 单点分析

**步骤：**

1. 在"经纬度"输入框输入坐标（格式：经,纬）
2. 点击"分析"按钮
3. 查看两行结果：
   - **区域属性** - 显示地区分类和POI密度
   - **地理特征** - 显示该点的地理特征标签

**示例：**

```
输入：116.3974,39.9093
结果：
区域属性：市区 (POI密度: 28/1km²) - 北京市海淀区中关村
地理特征：🛣️ 路边 | ✔️ 十字路口
```

### 批量分析

**工作流程：**

1. 准备包含地址的CSV文件
2. 点击"导入CSV文件"
3. 选择文件后自动处理
4. 点击"导出结果"保存文件

**输出文件包含字段：**

- `地址` - 原始输入地址
- `WGS84经度/纬度` - GPS坐标
- `GCJ02经度/纬度` - 中国坐标系
- `区域属性` - 地区分类信息
- **`地理特征`** - 该点的所有地理特征（新增）
- `状态` - 处理结果

## 🔍 特征识别规则

### 1. 路边识别

- 通过地址中包含的关键词识别
- 关键词：路、街、道、林荫道、环路、高速、快速

### 2. 十字路口识别

- 获取200米范围内的POI
- 分析POI分布方向（8个方向）
- 如果至少3个方向都有POI，则判断为十字路口

### 3. 高山识别

- 通过地址和行政区划关键词
- 关键词：山、峰、岭、岳、山区、高原、台地、爬山

### 4. 屋面识别

- 通过地址中的建筑相关关键词
- 关键词：楼顶、屋顶、天台、屋面、大厦顶、高楼

### 5. 水体识别

- 通过地址中的水体相关关键词
- 关键词：河、湖、海、江、溪、港、滩、水

### 6. 绿地识别

- 通过地址中的绿地相关关键词
- 关键词：公园、广场、绿地、花园、林地、草坪、森林

## ⚠️ 注意事项

### 准确性限制

- 特征识别基于地址关键词和POI分析
- 某些特殊情况可能误判（如"山水小区"被识别为高山+水体）
- 手动建筑物屋面难以准确识别

### 建议

- 对重要结果进行人工验证
- 结合地图查看确认特征
- 大批量数据可进行采样验证


```
✓ 密云北庄：乡镇 + 高山 → 考虑配送难度
```

**决策：** 优先在前两个位置建立网点

## 🔧 故障排除

### Q: 地理特征显示"无特殊特征"

**A:** 该位置不符合任何特征识别规则，这是正常的

### Q: 识别结果不准确

**A:** 地址信息不完整或关键词匹配问题，建议查阅地图确认

### Q: 十字路口识别不准

**A:** 基于POI分析，POI数据可能滞后或分布不均匀
