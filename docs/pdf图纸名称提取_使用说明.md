# PDF图纸名称提取 使用说明

## 1. 功能说明

脚本用于批量读取 PDF 图纸，提取每页图纸名称，并导出 Excel 结果。

对应脚本：
- tools/root_misc/pdf图纸名称提取.py

本次规则重点：
- 合并名称时自动去掉原始文件扩展名（例如 .pdf）。
- 新增对“基站天馈线新增示意图”的正常识别（非兜底追加）。
- 优化“面板图”与“设备布置平面图”优先级，避免将类似 DCPD10B面板图误判为主图名。

## 2. 运行环境

建议 Python 3.10+。

常见依赖（按需安装）：
- pandas
- openpyxl
- pdfplumber
- pymupdf
- pillow
- numpy
- opencv-python
- paddleocr（可选）
- pytesseract（可选）

## 3. 基本用法

### 3.1 批量处理目录

python tools/root_misc/pdf图纸名称提取.py --pdf-dir <输入PDF目录> --output-dir <输出目录>

说明：
- 输入目录会递归扫描 .pdf 文件。
- 输出目录会生成：
  - 图纸名称提取结果_时间戳.xlsx
  - 处理日志_时间戳.txt

### 3.2 单文件调试

python tools/root_misc/pdf图纸名称提取.py --single-pdf <单个PDF路径>

### 3.3 单图片调试

python tools/root_misc/pdf图纸名称提取.py --single-image <图片路径>

## 4. 输出结构说明

Excel 默认包含两个 Sheet：
- 原始提取：按文件记录每页提取结果（图纸1名称、图纸2名称...）。
- 合并后名称：按“文件名（去扩展名）+ 图纸名称”生成合并名称。

## 5. 脱敏与提交流程建议

为避免泄露业务信息，建议：
- 不提交原始 PDF、OCR 中间文件、提取日志、Excel 结果。
- 对外共享时使用脱敏目录名与样例数据。
- 提交前使用 git status 检查暂存文件，确保只包含代码与文档。

可参考最小提交范围：
- tools/root_misc/pdf图纸名称提取.py
- docs/pdf图纸名称提取_使用说明.md

## 6. 常见问题

1) 为什么批量跑完但结果看起来没变化？
- 脚本有历史结果复用策略。若输出目录已有历史结果，可能跳过已成功提取项。
- 建议改用新输出目录做对比验证。

2) 为什么出现退出码非 0，但有结果文件？
- 在部分终端环境下，GUI 相关收尾可能返回非 0。
- 如果 Excel 与日志已生成，通常不影响本次提取结果。

3) 仍出现“面板图”误判怎么办？
- 先用 --single-pdf 复测具体文件。
- 将有问题页的 OCR 文本样例补充到规则中再调整。
