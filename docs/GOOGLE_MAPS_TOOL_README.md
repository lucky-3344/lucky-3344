# Google Maps/Earth 坐标转换工具 - 使用说明

## 📋 功能概述

这是一个功能完整的地理坐标处理工具，集成了地址搜索、坐标转换、区域属性判断等功能。

### 核心功能

1. **地址搜索** - 通过地址名称获取精确坐标
2. **坐标转换** - WGS84 ↔ GCJ02 坐标互换
3. **区域属性判断** - 根据经纬度判断地区类型（市区/县城/乡镇/农村等）
4. **地图集成** - 一键打开 Google Maps/Earth/高德地图
5. **批量处理** - 支持 CSV 批量导入导出
6. **KML生成** - 创建 Google Earth 标记文件

## 🚀 快速开始

### 1. 单个地址查询

```
步骤：
1. 在"地址搜索"框输入地址（如：北京市海淀区中关村大街1号）
2. 点击"搜索"按钮
3. 若有多个匹配结果，选择正确的地址
4. 显示 WGS84 和 GCJ02 坐标
5. 点击相应按钮打开地图或复制 KML
```

**限制：** 搜索框支持最多84字节输入（中文占3字节）

## 🌍 区域属性判断

### 功能说明

根据经纬度判断该点在高德地图中的区域属性，返回以下分类之一：

| 分类 | 说明 | POI密度 |
|------|------|--------|
| **密集市区** | 一二线城市中心、商业密集区 | ≥30个/1km² |
| **市区** | 城市主要区域、人口稠密 | 15-30个/1km² |
| **县城城区** | 县级市中心、县城城区 | 8-15个/1km² |
| **乡镇** | 乡镇中心、人口较少 | 3-8个/1km² |
| **农村** | 农村地区、人烟稀少 | <3个/1km² |
| **郊区** | 开发区、工业区 | 特殊识别 |

### 使用方法

```
步骤：
1. 在"区域属性判断"区域的输入框中输入经纬度
2. 格式：经度,纬度（如：116.3974,39.9093）
3. 点击"分析"按钮
4. 结果显示：区域类型 + POI密度 + 行政区划信息
```

### 示例

**输入：** `116.3974,39.9093`（北京西二旗）
**输出：** `市区 (POI密度: 28/1km²) - 北京市海淀区中关村`

**输入：** `116.5023,40.0556`（怀柔山区）
**输出：** `农村 (POI密度: 2/1km²) - 北京市怀柔区`

## 📁 批量处理

### CSV导入格式

**文件要求：**

- 编码格式：UTF-8
- 分隔符：逗号（,）
- 第一行为表头

**示例文件：**

```csv
地址
北京市海淀区中关村大街1号
深圳市南山区科技园路1号
上海市浦东新区世纪大道100号
杭州市西湖区文一路258号
```

### 处理流程

```
步骤：
1. 点击"导入CSV文件"按钮
2. 选择包含地址的 CSV 文件
3. 等待处理完成（显示处理统计信息）
4. 点击"导出结果"保存处理结果
```

### 导出结果字段

处理完成后导出的 Excel 或 CSV 包含以下列：

| 字段 | 说明 | 示例 |
|------|------|------|
| 地址 | 原始输入地址 | 北京市海淀区中关村大街1号 |
| WGS84经度 | GPS坐标经度 | 116.3046 |
| WGS84纬度 | GPS坐标纬度 | 39.9926 |
| GCJ02经度 | 高德地图经度 | 116.3142 |
| GCJ02纬度 | 高德地图纬度 | 39.9892 |
| 区域属性 | 区域分类 | 密集市区 (POI密度: 42/1km²) - 北京市海淀区 |
| 状态 | 处理状态 | 成功/未找到地址/错误信息 |

### 导出格式

支持两种格式导出：

- **Excel (.xlsx)** - 推荐用于数据分析
- **CSV (.csv)** - 轻量级格式，通用性强

## 🗺️ 地图集成

### 打开方式

| 按钮 | 功能 | 要求 |
|------|------|------|
| **Google地图网页中打开** | 使用 WGS84 坐标打开浏览器 | 需要网络连接 |
| **Google Earth中打开** | 生成临时 KML 文件并打开 | 需装有 Google Earth Pro |
| **高德地图中打开** | 使用 GCJ02 坐标打开高德地图 | 需要网络连接 |
| **复制KML代码** | 复制 KML 标记代码到剪贴板 | - |

### KML文件说明

KML (Keyhole Markup Language) 是 Google Earth 的标准格式。

**使用方法：**

```
1. 复制 KML 代码
2. 打开 Google Earth Pro
3. 选择 文件 → 导入
4. 粘贴 KML 代码或选择导出的 KML 文件
5. 坐标点会显示在地图上
```

## ⚙️ 配置说明

### API密钥管理

该工具使用**高德地图 API**。默认已配置密钥，但建议为生产环境重新配置。

#### 方法1：环境变量（推荐）

```bash
# 设置环境变量
export AMAP_KEY="your_amap_key_here"

# Windows PowerShell
$env:AMAP_KEY="your_amap_key_here"
```

#### 方法2：配置文件

在程序目录下创建 `amap_config.json`：

```json
{
    "amap_key": "your_amap_key_here"
}
```

#### 获取密钥

1. 访问 [高德开放平台](https://lbs.amap.com/)
2. 注册账号
3. 创建应用获取密钥
4. Web 服务 API 权限必须包含：
   - 地理编码服务
   - 逆地理编码服务
   - 周边查询

## 📊 工作原理

### 区域属性判断算法

```
输入：经纬度坐标
    ↓
1. 逆地理编码（获取地址）
    ↓
2. 统计周边 POI 密度（1km范围）
    ↓
3. 分析地址结构和关键词
    ↓
4. 综合判断确定区域属性
    ↓
输出：区域类型 + 详细信息
```

### 分类规则

**规则1：** POI密度初步分类

- POI数 ≥ 30 → 密集市区
- POI数 15-29 → 市区
- POI数 8-14 → 县城城区
- POI数 3-7 → 乡镇
- POI数 < 3 → 农村

**规则2：** 地址关键词微调

- 包含"市辖区"、"城中" → 提升为市区
- 包含"县城"、"县" → 分类为县城城区
- 包含"乡"、"镇" → 分类为乡镇
- 包含"开发区"、"工业区" → 分类为郊区

**规则3：** 特殊区域识别

- 行政级别 + POI密度综合判断
- 县级区域POI < 20时可能是乡镇

## ⚠️ 常见问题

### Q: 搜索地址时显示"未找到该地址"

**A:**

1. 检查地址输入是否正确
2. 尝试使用更具体的地址（包含市、区、街道）
3. 某些偏远地区可能不被收录

### Q: 坐标转换有偏差

**A:**

- WGS84是GPS标准坐标系
- GCJ02是高德地图使用的坐标系
- 转换过程中会有 1-2 米的误差，这是正常的
- 其他地图应用可能使用不同坐标系

### Q: 批量导入很慢

**A:**

- API调用存在延迟，大批量(>100条)请等待
- 可分批处理减少单次等待时间
- 建议检查网络连接质量

### Q: "密集市区" 判断不准确

**A:**

- 区域属性判断基于POI密度，某些特殊区域(大型工业区等)可能误判
- 可结合返回的行政区划信息手动调整
- POI数据来自高德地图，可能存在滞后

### Q: Google Earth 打开失败

**A:**

1. 确认已安装 Google Earth Pro
2. 尝试手动导入：文件 → 导入 → 选择 KML 文件
3. 某些系统可能需要管理员权限

## 🔧 技术参数

### API限制

| 项目 | 限制 | 说明 |
|------|------|------|
| 单次请求超时 | 5秒 | 网络延迟可能导致超时 |
| 周边POI查询 | 最多50条 | 1km范围内最多返回50个结果 |
| 地址输入长度 | 84字节 | 中文占3字节 |
| 坐标精度 | 小数点后6位 | 精确到1米以内 |

### 坐标系统

```
WGS84（World Geodetic System）
  ↔ 转换 ↔
GCJ02（高德/腾讯地图坐标系）
```

转换误差范围：1-2 米（正常范围内）
